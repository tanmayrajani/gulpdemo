So, WHY USE GULP?
Gulp solves the problem of repetition.
There are a lot of repetitive tasks involved with creating and deploying a front-end application.
Gulp helps us automate them.
Automating repetitive tasks = more time to do non-repetitive tasks = more productivity.
Think of gulp as a factory

It takes files and turns them into other files with very minimal effort.
WHY U NO COME STRAIGHT TO THE POINT?
Gulp does nothing but provide some streams and a basic task system.
Gulp has only 5 functions you need to learn.

You're just 10 minutes away from becoming a gulp expert.
gulp.task(name, fn)
It registers the function with a name.
You can optionally specify some dependencies if other tasks need to run first.
gulp.src(filepattern)
This returns a readable stream.
Takes a file system pattern and starts emitting files that match.
This is piped to other streams.
gulp.dest(folder)
This returns a writable stream.
File objects piped to this are saved to the file system.
gulp.watch(filepattern, fn)
Runs a function when a file that matches the filepattern changes.
Instead of function, we can specify array of tasks as well and it would run mentioned tasks instead.
gulp.run(tasks...)
Runs all tasks with maximum concurrency
Oh.. and one more thing!
We can use ".pipe()" to send one program's output as another's input.
